Creating your plugin from an existing template#

Note

This content is adapted from notes I (@gregcaporaso) put together as a temporary ‚Äú5-minute plugin development tutorial‚Äù. This section will be adapted to build from a cookiecutter project template, replacing many of the manual steps required here.

  1. Nick Bokulich‚Äôs lab maintains a template QIIME 2 plugin that is a great start for your plugin. You can find this here: bokulich-lab/q2-plugin-template. There‚Äôs a link on that page to ‚ÄúUse this template‚Äù. Click that, and GitHub will fork that repository and you can start working on your plugin.

  2. To work on the plugin, first install a QIIME 2 development environment (see Set up your development environment).

  3. Activate your development environment.

  4. Then, clone your new plugin repository and run through the steps in the plugin template README. I created a test plugin on 22 February 2024 and confirmed that it all works. Here are the changes I made when I did this step (and the second set of changes I made, when I realized I missed some occurences).

    • Tip: Rename the q2_plugin_name directory using git mv q2_plugin_name <new plugin name>.

    • Tip: It‚Äôs easiest to replace q2-plugin-name, q2_plugin_name, plugin_name, and plugin-name in the files using a global search/replace - there are many occurences.

  5. From the top-level directiory of your plugin repository, install your plugin in developer mode by running pip install -e ..

  6. To confirm that your plugin is now available, run qiime info. You should see your new plugin in the list of available plugins. (If you get an error that says something like ModuleNotFoundError: No module named 'q2_plugin_name', you didn‚Äôt replace all instances of q2_plugin_name in the template repository.)

  7. Next, add an action. Let‚Äôs create one that that takes a FeatureTable[Frequency] (i.e., a feature table with count data) as input and generates one as output. I created an action in my test plugin with this signature - all it does is return a copy of the input table. Copy this into your plugin.

  8. Then, run qiime dev refresh-cache, and then qiime --help. You should see your new plugin in the list of plugins. If you call help on it, you should see your new action. You should be able to run the duplicate-table action in your plugin as you can for any other QIIME 2 plugin.
